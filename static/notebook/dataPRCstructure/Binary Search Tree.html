Binary Search Tree

• 대소관계규칙 : left child < parent < right child
        → input 순서에 따라 굉장히 skewed 될 수 있음
   
            

        → 그래서 tree 높이가 항상 최소가 되도록 하는 변종 bst 들이 존재 (예: Red-Black Tree)
• 대소관계에서 비롯되는 후보의 수를 (평균적으로) 절반씩 줄여나가는 방법 때문에, fast look up이 장점
    ◦ 사실 look up, traverse 같은 경우는 array 로 구현한 binary search 가 편하지만
      array의 선형적인 구조의 제약으로 인해 insert 와 delete 가 bst에 비해 불편해서
      추가제거가 빈번하다면 bst가 바람직할 수 있음 

• Insert - O(logn)
    훑고 그냥 넣으면 되기 때문에 간단함. traverse 를 하기 위한 시간 average O(h) 
    (worst는 완전 skewed tree 인 경우이므로 O(n))

• Delete - O(logn)
    살짝 복잡한데 방법이 한가지만 있는게 아님
    학교에서 배운건 다음의 케이스에 따라 delete 하는 것
    (1) delete 되는 node의 child = 0
    (2) delete 되는 node의 child = 1
    (3) delete 되는 node의 child = 2
        ① D ⟷ E
          D의 값을, 대소관계규칙을 깨지 않을 subtree 내 노드 E을 찾아 replace
              ◦ in-order predecessor (L subtree 에서 최댓값) 또는 - C
              ◦ in-order successor (R subtree 에서 최솟값)       - E
        ② (if E child) E ⟷ E.Child
            우리가 E를 찾는 방법때문에, E의 child는 최대 한개이므로 바로 replace 가능
