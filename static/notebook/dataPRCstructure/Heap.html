Heap

구조적 제약 → 특징
    • 대소관계규칙 : 부모 node > 자식 node (min heap인 경우 반대)
        → max/min을 찾는데 최적화된 형태의 이진 트리 (항상 max/min이 root에 있으므로)

        

    • 모양규칙 : 노드의 개수만으로 heap의 모양이 정해짐 → list등으로도 쉽게 구현 
        ◦ 마지막레벨을 제외한 모든 tree는 채워져 있어야 한다
        ◦ 마지막레벨에서 왼쪽부터 채워져 있어야 한다

• Insert - O(logN) : 모양규칙, 대소관계규칙을 만족시키도록
    ① root와 새 원소를 비교한다.
    ② [모양규칙]을 우선 만족시키기 위해, 일단 끝에 넣는다.
    ③ [대소관계규칙]이 만족될 때까지 조정한다.

    

• Delete - O(logN) : 최대최소원소 꺼내기
    ① root 를 지운다.
    ② [모양규칙]을 우선 만족시키기 위해, 어차피 지워질 운명인 마지막 leaf node를 root로 옮긴다.
    ③ [대소관계규칙]이 만족될 때까지 조정한다.
