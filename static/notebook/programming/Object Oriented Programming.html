Object Oriented Programming

• Data 와 그 데이터를 변형시키는 Procedure 를 모두 가지고 있는게 Object를 바탕으로한 디자인
• 이를 분리 시켜 생각하는게 Procedural Language

장점
    ① Encapsulation
    ② Polymorphism 다형성 duck typing
        • base class를 재정의하는 overriding를 통해 same face, different form
        • 같은 interface지만 어떤 object가 runtime에 주어졌느냐에 다른 behavior을 발생하도록 함

    ③ Abstraction
        data 뿐만 아니라 behavior 까지, 완벽한 abstraction

    ④ Inheritance
        우리가 모델링하려고 하는 세상의 object 들은 유사한 경우가 많다. (예: 악기 – 관악기, 현악기)
        1000개의 악기가 있다고 1000개의 object를 만드는 것은 비효율적이다.
        나름의 공통점을 기준으로 class hierarchy를 만들어서, organize 하자.
        가장 포괄적인 object를 parent라 두고,
        그 밑에는 ‘parent에 원래 있던 것(inheritance) + parent와의 차이점’으로 재정의하자
