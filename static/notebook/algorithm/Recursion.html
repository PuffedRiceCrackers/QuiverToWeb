Recursion

• 자기자신으로 정의될 수 있는 문제
    Problem that is defined in terms of itself/its type (예 : F(n)=F(n-1)+F(n-2))
    (예) Divide and Conquer (subproblem 균등분배), Dynamic Programming (memoization)
def fib(n): ・・・・・・・・・・・・①
if n<1: ・・・・・・・・・・・・②
return n
else:
return f(n)+f(n-1) ・・・・・③
방법
    ① subproblem arg 정하기
        • tree에서 해당 subproblem 의 위치를 잘 드러낼 수 있고
        • 문제의 재귀적정의가 분명할 수 있도록정해야함(누가 누구를 부르게 되는지?)

    ② edge 답이 바로 나오는 구간

    ③ merging rule
        • subproblem 의 해를 구했다고 가정하고, 이를 통해 problem 의 해를 어떻게 구성할 수 있는지?        
          (예) 피보나치의 경우엔 f(n-1) + f(n-2) 를 더한다가 일종의 rule 이 될 거고, 문제마다 이는 다를 것

Tip
    • 일반적인 경우 함수는 return 값이 있어야 함
    • edge case 를 가장 마지막에서 잡아야 코드가 짧아짐
      (예) xbbww 라고 주어졌을 때, edge를 dic(pic,1)=bbww로 보는 것이 아니라
          div(pic,1)=b,…,div(pic,4)=w 처럼 최소단위로 쪼갤 때 생각하기 간단함 
    • func 의 return 값을 간과하고 merging rule 을 만들지 않도록 주의한다.
        (예) quickSort 를 다음과 같이 짜려고 한 적이 있었는데,
         quickSort(A, start, end)  는  return 소트된 A  로 정의했기 때문에
        🐛 부분마다 부분소트된 A가 통째로 들어가다 보니 최종답이
     [5, 9, 30, 34, 72, 9, 5, 9, 30, 34, 72, 34, 5, 9, 30, 34, 72] 이렇게 나와버림
def quickSort(A, start, end):
if start >= end:
return A
else:
# A를 파티션한다
answer = []
answer.extend(quickSort(A, start, wall - 1)) ・・・・🐛
answer.append(A[wall])
answer.extend(quickSort(A, wall + 1, end)) ・・・・🐛
return answer
