heapq, PriorityQueue

• min, max를 구하는 데 특화된 자료구조 (기본 library 는 min만 지원)
• 사실 Priority Queue의 내부구조는 heapq라고 함
    - 다만 pq가 문법상 더 쓰기 편하고, heapq와 달리 thread safety가 보장된다고

heap
    문법이 다 method 형식이 아닌 func 형식이라 헷갈림

    ※ 내가 생각하는 문법 (틀림)
from heapq import heapify, heappop, heappush
a = heapify([1,2,3])
a.heappop()
a.heappush(4)
    ※ 실제 문법
from heapq import heapify, heappop, heappush
a = [1,2,3]
heapify(a) # return값이 있는 모듈이 아니므로 a = heapify(a) 도 안된다는 거.
heappop(a)
heappush(a,4)
len(a)
    • 쓰기도 불편한 heapq지만 몇가지 장점이 있는데
        ◦ 기본적으로 list 임 : list의 op들을 쓸 수 있음 (heap[0] 으로 root 접근, len(heap) 등…)
        ◦ init 을 한꺼번에 할 수 있음
        ◦ priority queue보다 thread safety와 관련한 overhead가 없어서 더 빠름
priority queue
    • 용어가 특이 (push, pop 대신 .put(), .get())
    • 사용상 와닿는 가장 큰 특징은 한꺼번에 init이 안되고 하나하나 .put() 해줘야 한다는 점    
from queue import PriorityQueue
pq = PriorityQueue()
pq.put((1, "one"))
pq.put((3, "tree"))
pq.put((2, "two"))
a = pq.get() # (1, "one")
b = pq.get() # (2, "two")
c = pq.get() # (3, "tree")
pq.qsize() >> 0
이런 식으로 한꺼번에는 못 만드니 주의할 것
a = PriorityQueue([(1,'가'), (2,'나')])
