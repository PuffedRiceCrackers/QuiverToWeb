method 의 종류

기본 : class 나 obj 를 arg 넘겨받을 수만 있으면 class/obj 수정은 누구나 가능함 (private이 아닌 이상)
근데 class, static, instance method 는 default arg가 다를 뿐이지

• Class Method @classmethod
    기본적으로 class를 넘겨받고, 추가적으로 object도 넘겨받을 수 있음
• Static Method @staticmethod
    아무것도 넘겨받지 않지만, class/obj를 넘겨받을 수 있으니, 이들을 수정도 가능함
    사실상 class/obj를 통해서만 접근 가능하다는 점빼면 일반 func랑 똑같음
• Instance Method 데코레이터가 없는 함수
    기본적으로 self (object)를 넘겨받고, 추가적으로 class도 넘겨받을 수 있음

※ 기본적으로 cls, self가 넘어가는 경우는, 함수 call 할 때 이를 넘겨주면 안 됨
class myClass:
classVar = 0
@classmethod
def classFunc(cls):
print('class func')
@staticmethod
def staticFunc_a():
print('static func a')
@staticmethod
def staticFunc_b(cls, self):
print('static func b')
cls.classVar += 1
self.instanceVar += 1
def __init__(self, a):
self.instanceVar = a
def instanceFunc(self):
print('instance func')
obj = myClass(0)
myClass.staticFunc_a() # ⭕️ class 에서 static func 접근
obj.staticFunc_a() # ⭕️ object 에서 static func 접근
staticFunc_a() # ❌ sttatic 을 direct로는 접근 못함
myClass.staticFunc_b(myClass, obj) # ⭕️ class 에서 static func 접근
obj.staticFunc_b(myClass, obj) # ⭕️ object 에서 static func 접근
