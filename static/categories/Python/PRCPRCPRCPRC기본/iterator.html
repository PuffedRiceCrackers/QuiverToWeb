iterator

• 전체에서 한개씩 뽑아낼 수 있는 것
• iterator 의 유일한 행동은 next()
• next() 빼곤 사실상 iterable 과 같고, iterable 이 들어가는 모든 곳에 들어갈 수 있음
a = iter([1,2,3])
>> type(a) # list_iterator
• lazy (실행되기 전까지는 메모리상에 올라와있지 않다가, 실행될 때만 메모리상에 올리는 것)
    전체를 한번에 memory 에 올리지 않으며, next() 할 때만 올리기 때문에 메모리관리에 좋음 
    (빅데이터에 유용)(속도는 조금 느림)
    👏🏻 근데 나중에 한개씩 바뀔려면 뭘로 바뀔지도 어딘가 저장되어 있어야 하는거 아닌가 

• functional programming 과 관련이 있는 이유
    functional programming 에서는 여러개를 한번에 동시에 처리하는 것이 일반적인데,
    이를 위해 하나씩 뽑아서 처리하는게 필요하기 때문에

• 뽑아내고 바뀜 
a = iter([1,2,3]) # iter()를 통해 iterator로 바꿀 수 있음
next(a) # 1
list(a) # [2,3]
next(a) # 2
list(a) # [3]
next(a) # 3
list(a) # []
• cycle 이라는 iterator 도 있음
from itertools import cycle
a = cycle([1,2,3])
next(a) # 1
next(a) # 2
next(a) # 3
next(a) # 1
• 파일 읽는 것도 한꺼번에 올리면 비효율적이라서 내부적으로 모두 iterator generator 로 만들어져있음
