mutable / immutable

• mutable, immutable 은 니가 알고 있는 그대로임. 바꿀 수 있느냐 못 바꾸냐.
• python 은 상수가 없어서 (= 재할당 불가능), immutable 로 데이터를 보호하는 장치가 필요했음

    ① Mutable objects:
        list, dict, set, byte array, user-defined class
        값의 첫번째 주소
        값을 바뀌어도 메모리 주소는 바뀌지 않음 (자기자신의 크기만 바뀜)    
    ② Immutable objects:
        int, float, complex, string, tuple, frozen set, bytes

※ Copy / Func에 arg로 넘길 때
  Mutable : pass by reference, memory address가 복사됨
            실제 함수를 거치고 나면 원래 object 가 바뀌어 있음
  Immutable : pass by value, value 가 복사됨
              실제 함수를 거치고 나도, value만 넘어가서 새로운 copy를 변형하는거라 원래 object 는 그대로임
def function(array, integer):
# array 를 변형함
# integer 를 변형함
array = [1, 2, 3]
integer = 1
function(array, integer)
print(array) # 원래 array가 변형됨
print(integer) # 원래 integer는 변형되지 않음
• Mutable object 의 value 를 copy 하고 싶다
