re.findall(pattern, string)
    • string에서 해당 pattern에 해당하는 애들만 추출해서 반환
    • pattern을 줄 때 ()로 묶으면, result도 분리되어 나옴
    • tag를 고를 때 유용 : (.*)가 아니라 (.*?) 여야 됨을 기억
import re
re.findall('<h3 class="LC20lb">(.*?)</h3>', '<h3 class="LC20lb">멕시코</h3>')
>> ['멕시코']
re.findall("\d{8} \d{1,2}시", "19243252 23시")
>> ["19243252 23시"]
re.findall("(\d{8}) (\d{1,2}시)", "19243252 23시")
>> [('19243252', '23시')]
re.compile()
    pattern를 regex obj로 만듦
re.sub(pattern, replace, target string)
    string에서 pattern을 찾아, replace
pattern = re.compile("(a+)(.*)(b+)") # pattern을 reg obj로 바꿈
re.sub(pattern, 'xxx', 'physics aaakbbb philosophy')
pattern.sub('xxx','physics aaakbbb philosophy')
>> 'physics xxx philosophy'
pattern.search(target string)
    match obj 를 반환함 (이걸 어찌 써먹는진 모르겠지만)
pattern = re.compile("(a+)(.*)(b+)")
pattern.search("a1bk")
>> <re.Match object; span=(0, 3), match='a1b'>
기본적으로 정규식의 모든 룰은 ()하지 않는 이상 하나의 char에 적용되는 듯 함
————————————————————반복————————————————————

*
  0~ 반복

+
  1~ 반복

{m}, {m,}, {m,n}
  m번, m번 이상, m에서 n번까지 반복
  • 굳이 () 붙이지 않아도 1 char에 바인딩 : waz{2,}up 이면 z에만 규칙적?

?
  optional char
  • files? 
    file도 잡히고 files도 잡힘

————————————————————type————————————————————

.
  아무거나

\d
  0~9

\D
  0~9가 아닌 것

\w
  a~z,A~Z,0~9

\W
  a~z,A~Z,0~9가 아닌 것 (한글, 특수문자, punctuation...)

\s
  white space
  space (␣), the tab (\t), the new line (\n), and the carriage return (\r)

\S
  any none white space

————————————————————or/nor————————————————————

[xyz]
  or (x,y,z 중 하나면 true)

[^drp]
  ^d or ^r or ^p (d,r,p만 아니면 true)
  • [] 식으로 써야지 (^d) 이런 식으론 못쓰는 듯 함 

————————————————————시작과 끝————————————————————

^Mission
  Mission 으로 시작
  (O) Mission: successful
  (X) Last Mission: unsuccessful
  (X) Next Mission: successful upon capture of target To be completed

successful$
  successful로 끝남
  (O) Mission: successful
  (X) Last Mission: unsuccessful
  (X) Next Mission: successful upon capture of target

^Mission: successful$
  정확히 Mission: successful
  
———————————————————grouping—————————————————————

()
  grouping

  패턴 : (\d+)x(\d+)
  예시 : 1280x720
  >> 1280x720, 1280, 720이 모두 잡힘

  ([a-zA-Z]+ (\d+)) idkw, 띄어쓰기 해야됨
  이거 덕분에 (.*) capture all 을 할 수 있음

————————————————————condition————————————————————

(abc|xyz)
  abc 거나 xyz 거나
  • I love (cats|dogs)

———————————————————기타 ————————————————————

\
  escape. 기호로서의 ., ? 가 아니라 실제 . ? 를 포착하고 싶을 때

  \? : "?"
