Cascade

• ForeignKey 에서
    ◦ 내 기준에서 더 쉽고 확실
    ◦ 실제로 Person.id 업데이트시 같이 업데이트되고,
      어떤 person을 지우면, 딸려 있던 cellphone도 같이 없어짐을 확인함
# 1 : Person
class Person(base):
...
# N : Cellphone
class Cellphone(base):
...
person_id = Column(INTEGER, ForeignKey('person.id', ondelete="cascade", onupdate="cascade"))
• relationship 에서
    Cascade의 default는 save-update, merge로 되어있음 [doc]
class User(Base):
addresses = relationship("Address", cascade="save-update, merge, delete, delete-orphan")
    •  save-update  (default)
        어떤 object 가 session 위에 올라가 있으면,
        그것과 relationship 으로 엮여있는 모든 object 역시 session 으로 올려줌

    •  delete ​
        ◦ delete 가 없었더라면, address 에서 user의 fk를 NULL로 하는 게 기본 op인데
        ◦ delete 가 있으므로 user 가 지워지면, user랑 연결되어 있던 address도 지워짐    
    •  delete-orphan 
        ◦ parent 가 delete 될 때 뿐 아니라, parent-child 간 de-association 이 발생할 때도 지우라는 말
        ◦ 일단 1:N relationship 일 때만 쓰는 것
