Struct, Class

• struct class는 모든면에서 동일하나 두개정도가 크게 다름

① Struct
    • no inheritance
    • value types 으로서 heap 에서 살지 않고, 다양한 상황에서 그냥 copy 됨
        - 그러나 copy on write (변경될때만 copy) 정책을 가짐
        - int, str, dict… 모든것이 다 struct임
struct Card {...}
class Concentration
{
...
init(numberOfPairsOfCards:Int)
{
let card = Card(identifier: identifier)
let matchingCard = card ⭐️
/*let matchingCard = Card(identifier: identifier)
대신에 위처럼 간단하게 쓸 수 있는데, card 라고만 해도 전체가 copy 될 것이기 때문 */
}
}
    • free init 을 쓸 수 있긴 한데 그러려면 pre-init 되있는 것들도 다 init을 해야한다?
// Concentration.swift
struct Card
{
var isFaceUp = false
var isMatched = false
var identifier:Int
}
// Concentration.swift
class Concentration
{
...
init(numberOfPairsOfCards:Int)
{
/* Card 라는 struct에 init이 없는데 free init을 사용하려는 상황 */
let card = Card(isFaceUp: false, isMatched: false, identifier: identifier) ⭐️
}
}
• 만약 self 를 변경할 가능성이 있는 func 가 안에 있을 시 mutating 이라는 표시를 해야함
    - value type 이기 때문에, 변경될 때만 copy가 되는데, 그 변경이 언제되느냐를 나타내 주는 것임
    (var 같은 경우는 원래 변하는 애니까 안해도 되고)
struct Concentration
{
private(set) var cards = [Card]()
private var indexOfOneAndOnlyFaceUpCard:Int?{...}
mutating func chooseCard(at index:Int){
if !cards[index].isMatched{
if let matchIndex = indexOfOneAndOnlyFaceUpCard, matchIndex != index{
if cards[matchIndex].identifier == cards[index].identifier {
cards[matchIndex].isMatched = true // self 변경
cards[index].isMatched = true // self 변경
}
cards[index].isFaceUp = true // self 변경
} else {
indexOfOneAndOnlyFaceUpCard = index // self 변경
}
}
}
}

② Class
    • inheritance 문법 : class A:B (A가 B를 상속한다)
    • reference types
        - heap 에 있으면서, pointer만 주고받는 (하나의 object 에 대해 여러개의 ptr 가 존재할 수 있음) 
        - 참고로 swift에서 reference type은 class와 closure 뿐임
    • free init
        - class내부의 var만 모두 init이 되있으면 free init을 받는다
// Concentration.swift
class Concentration
{
var cards = [Card]()
func chooseCard(at index:Int){}
}
// ViewController.swift
class ViewController: UIViewController
{
var game = Concentration() ⭐️ /* 에러없이 init이 잘 됨*/
}
