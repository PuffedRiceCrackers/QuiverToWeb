Function type 과 Closure

Function 자체가 하나의 타입임
예를들어 “Arg를 Double로 받고, Double을 return하는 Function” 이라는 type은 다음과 같이 정의할 수 있는 것
var operation : (Double) -> Double
operation = sqrt // sqrt가 위와같은 조건을 만족하는 func땜에 legal
let result = operation(4.0)
하나의 func 를 assign 하는 syntax 를 보면 다음과 같이 많은 라인이 필요함
var operation : (Double) -> Double
func changeSign(operand:Double)->Double { return -operand }
operation = changeSign
Closure
    이라는 문법을 사용하면 더 간단하게 만들 수 있음 (first curly brace 를 앞으로 빼내고, 그 자리에 in을 씀)
var operation : (Double) -> Double
operation = { (operand:Double) -> Double in return -operand }
여기서 생략 가능한 것들이 몇가지 있는데, 이를 적용해서 다음과 같이 극단적으로 간단하게 나타낼 수 있음
    - operation 의 arg type 과 return type 을 정의할 때에 의해 type inference 할 수 있음
    - operand 라는 이름이 필요 없음 ($0, $1, $2… 로 대체 가능)
    -  in 이나 return 이라는 문법적 장치
        → operation = { (operand : Double) -> Double in return -operand }
var operation : (Double) -> Double
operation = { -$0 }
closure 은 다양한 곳에서 유용하게 쓰일 수 있다

① function이 function을 arg로 받을 때 
/* map 은 function 을 arg 로 받아서, 그 function 을 각 array의 element에 적용해,
그걸로 새로운 array를 만드는 method 임 */
let primes = [0.1, 0.2, 0.3, 0.4]
let negativePrimes = primes.map({ -$0 })
let invertedPrimes = primes.map(){ 1.0/$0 } // last arg 가 closure func일 경우, 밖으로 뺄 수 있음.
let stringPrimes = primes.map { String($0) } // 만약 arg가 1개라면 아에 ()를 없앨수도 있음
② property initialization
var someProperty : Type = {
// construct the value of some property
return <the constructed value>
}()
// () 이걸로 인해 바로 실행을 하게 됨
