Internal, Private, Public

• 여러명이서 하나의 프로젝트를 다룬다 할 때 다른데서 내 var 지우고 이러면 내 code가 통째로 안 돌아가고 이럴 것이므로
• 일단 private을 쓰고, 공개할만한 게 있는지를 봐서 public/internal 등으로 돌리는게 바람직
• internal implementation 인 것들은 감추기
• 대략 오픈정도가 internal > fileprivate > private(set) > private

internal
    default. unlimited
private
    only callable within this object
private(set)
    readable outside this object, but settable only within this object
fileprivate
    동일한 file 안의 서로 다른 object(class)들끼리 공유가

public
    frameworks only
    can be used by objects outside my framework
Open
    frameworks only
