protocol

• a collection of declaration (method/property)로, API를 좀 더 flexible/expressive하게 만들어줌
• Controller 와 View 간의 blind communication 에 (delegation), dictionary 의 key를
  unique하게 enforce할 때, Dictionary 에서 key 를 지정할 때 
• 일종의 type 임
• swift 에서는 어떤 protocol 을 쓰겠다고 하면, 그 안의 모든 걸 써야함 (obj-c는 선택이 가능했음)
• 만약 protocol 에 init 이 있고, 그 protocol 을 쓰는 게 class 라면 init 앞에 required 라고 붙여줘야 한다
    - subclass
protocol someProtocol : inheritedProtocol1, inheritedProtocol2{
var someProperty : Int { get set }
func aMethod(arg1:Double) -> Int
mutating func changeIt()
init(arg : Int)
}
• protocol 의 선언 : declaration of vars/functions 
protocol Moveable {
mutating func move(to point:CGPoint)
}
• class/struct/enum 등은 해당 protocol 을 사용하겠다고 함
    - 이 때, 각 실제 implementation (예: func move()) 은, 알아서 다를 수 있음 (car 에 맞는 move, shape 에 맞는 move)
class Car : carSuperClass, Moveable{
func move(to point:CGPoint){...}
func changeOil()
}
struct Shape : shapeSuperClass, Moveable{
mutating func move(to point:CGPoint){...}
func draw(){...}
}
• 다음과 같이 활용되는 걸 보면, 지극히 type 처럼 작동함을 알 수 있음 (왜냐면 type이 맞으니까!)
let prius:Car = Car()
let square:Shape = Shape()
// thingToMove는 Moveable type이고, Prius 는 Moveable Protocol 을 사용하므로 가능
// prius 와 square 는 매우 다르지만, 둘다 Moveable이기 때문에 같은 array안에도 들어갈 수 있게 됨
var thingToMove:Moveable = Prius
let thingsToMove:[Moveable] = [prius, square]
thingToMove.move(to:...) // legal - prius(car) 의 move()를 쓸 것
thingToMove.changeOil() // illegal - thingToMove는 Moveable 타입이지 Car 타입은 아님
thingToMove = square
thingToMove.move(to:...) // square(Shape)의 move()를 쓸 것
func slide(slider: Moveable){ // Moveable type 을 arg로 받으므로
let positionToSlideTo = ...
slider.move(to:positionToSlideTo)
}
slide(prius) // Moveable protocol을 사용하는 property들이 arg로 들어갈 수 있음
slide(square)
func slipAndSlide(x : Slippery & Moveable)
slipAndSlide(prius)
• 참고로 선언 시 class 라고 붙여주면, 해당 protocol 은 class 밖에 못 쓰게 됨(그러면 mutating 이런걸 쓸 필요도 X) 
// 잘 쓰이진 않음
protocol someProtocol : class, inheritedProtocol1, inheritedProtocol2 { ... }
