strong, weak, unowned

Automatic Reference Counting
    아무도 해당 대상을 가리키고 있지 않으면, 
    그 대상을 heap 에서 자동으로 지움 (이건 garbage collection이 아님)
• strong
    어떤 heap object를 가리키는 strong pointer가 하나라도 있으면, 그 object는 heap 에 머무름

• weak
    이 heap object를 가리키는 strong pointer가 있을 때만, heap 에 둬라
    (strong pointer가 있을 때만, 나는 이 object에 관심있다) 
    optional pointer to reference type에만 사용 (예:outlet)

• unowned
    이 대상이 만약 heap 에서 사라지면 (strong pointer가 없어지면)
    나도 얘를 가리키지 않을테니, 나를 reference count하지 말아라 
   (memory cycle 문제 풀때만 사용 - 서로 pointing each other 해서 둘다 heap 에 있는데, 아무도 사용되지 않음)
