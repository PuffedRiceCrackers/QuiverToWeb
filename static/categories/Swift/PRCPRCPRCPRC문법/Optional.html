Optional

• 파이썬에서도 메모리 ‘칸’만 할당하는 건 불가능 했음
  swift 에서도 비슷하다면 비슷한데, python과 달리 strongly typed language 기 때문에
var a:String = nil
  "나중에 String 을 넣을 건데, 지금은 nil 이야” 이런게 불가능함
• 그래서 값이 없을 때에 대한 장치로서 Optional 이란 타입을 만들었음
• 두개의 state만을 가짐
    set
        ! 했을 때 associated data를 가짐
    not set
        ! 하면 에러뜰 것임
var a:Int? = 1 // set
var b:String? // not set - nil
var c:Double? // not set - nil
• !
    unwrap. optional type 의 associated data를 가져오기 (안그러면 Optional(1) 이런식으로 반환됨)
    nil 이 아닌 게 확실할 때만 (nil 인데 !하면 crash남)
print(a!) // 1
• ??
    optional 의 state 을 기준으로 플로우를 만들고 싶은 경우
return emoji[card.identifier] ?? "?" //not set 이면 “?”를 리턴, set이면 associated data 리턴
var a:Int?
var b = a ?? 0 //not set이면 b=0, set이면 b=a의 associated data
• dictionary 에서 값을 반납할 때 기본적으로 Optional 임
  못 찾을 경우 nil을 반납해야 되기 때문
var a = [1:"a", 2:"b", 3:"c"]
var b = a[1]
print(b) // Optional("a")
Optional 은 사실상 이렇게 생긴 enum임 
enum Optional<T>{
case none
case some(<T>)
}
